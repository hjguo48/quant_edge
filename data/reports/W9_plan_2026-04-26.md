**目标**：W8 跑了 11 windows (W-1 ~ W9)，W9 补完 W10/W11 + 早期 W-2/W-3（如果数据范围支持）

**Gate**：在 11 重叠 windows 上**无 regression**（mean IC 不能跌超过 5%）；新加的 2 windows 至少 1 horizon 仍正且 t > 1.5

**输出**：`data/reports/walkforward_v9full9y_{1d,5d,20d,60d}_ridge.json`

### W9.2 — Date-level regime label table 构建

**目标**：每 trade_date 一行，含 ex ante macro covariates，作为 weak-window gating classifier 的训练样本

**实施**：写 `scripts/build_regime_table.py`：
```
列:
- trade_date
- vix_level, vix_zscore_60d, vix_zscore_252d
- breadth_pct_above_20dma, breadth_pct_above_50dma
- market_vol_20d (SPY rolling vol)
- market_drawdown_60d (SPY peak to current)
- return_dispersion_20d (SPY cross-sectional std of forward return)
- yield_curve_inv (10Y-2Y)
- credit_spread (BAA-10Y)
- regime_label (TBD via downstream classifier)
```

**约束**：所有 covariates 严格 PIT — 用 trade_date 当天可见的 macro 数据，不能用 forward-looking

**资源**：从现有 `macro_series_pit` 表 + `stock_prices` SPY 数据，~1 min 跑完

**输出**：`data/features/regime_table_2016-2025.parquet`

### W9.3 — Weak-window 诊断 + gateable vs regime break 决策

**目标**：识别哪些 W8 weak windows 是"可 gating 的"vs"不可预测 regime break"

**实施**：
1. 拉 W8 60D Ridge per-window IC（W0=-0.007, W5=+0.011, W6=+0.008 是 weak）
2. 对每 weak window，看其 test 区间的 regime covariates 分布
3. 找出 weak windows 的共同 ex ante 特征（高 VIX？低 breadth？yield curve inverted？）
4. 拟合 simple monotonic rule 或 logistic regression on **date level**：
   - X = regime covariates
   - y = sign(daily_ic) 或 daily_top_decile_excess
5. 在 historical data 上 backtest gating rule：weak regime → 降仓 50% 或中性，strong regime → 满仓
6. 记录哪些 weak windows 是 "ex ante 可识别"（gateable），哪些是 "ex post 才知道"（regime break，接受损失）

**Gate**：至少 1 个 weak window 找到 ex ante regime 先验，且 OOS gating 后该 window IC 转正（或 top_decile 转正）

**输出**：`data/reports/weak_window_diagnosis_2026-04-26.md`

## W9 Verdict 标准（Codex 修正版）

不再用"13-window IC > 11-window W8" — 因为评估集不一样，比较没意义。

**新标准**：
1. **稳定性**（必须）：W8 11 重叠 windows 上无 regression（mean IC 跌幅 ≤ 5%）
2. **扩展性**（必须）：完整 13 windows 至少 1 horizon 仍正且 t > 2
3. **Gateability**（必须）：ex ante regime rule OOS 改善 ≥ 1 weak window，不显著伤害 strong windows
4. **Minute-proxy**（bonus，非必须）：2 个 viable features 至少 1 个全宇宙 audit PASS
