## W12-0 必做的 3 个 patch

### Patch 1: Greyscale runner — raw score mode

`scripts/run_greyscale_live.py`:
- 加 bundle field: `score_space: "raw"`
- 单 model + raw_score → skip `combine_current_predictions(normalized_predictions, ...)` 的 z-score
- 直接用 raw ridge predictions 给 `build_score_weighted_portfolio()`

**理由**: W10 champion 用 raw ridge scores. 现 greyscale z-scores → silently 改变策略.

### Patch 2: Greyscale runner — bundle universe filter

`scripts/run_greyscale_live.py`:
- 加 bundle field: `eligible_universe_path`
- 运行时 universe = `get_active_universe(...) ∩ eligible_universe_snapshot`

**理由**: champion 训练在 frozen 503 snapshot 上, 当前 active SP500 可能含 post-2025 加入的成员 (training set 外).

### Patch 3: Greyscale monitor — single-model awareness

`scripts/run_greyscale_monitor.py`:
- if active model count `< 2`: skip `model_consistency_above_threshold`

**理由**: 当前 monitor 假定 multi-model fusion, 单 model 会 trivially fail.

- `--as-of <historical date>`

**Pass criteria**:
- bundle validates
- `portfolio_scheme = score_weighted`
- 0 critical layer1/2/3/4 alerts
- holding count ≥ 20
- 非空 target weights
- 第二次 dry-run 干净复用 previous weights

**不能 over-interpret**:
- 第一次 dry-run turnover 不能跟 backtest 比较
- Realized IC 在 prep 阶段不可用
- Dry-run 验 wiring, 不验 economics
