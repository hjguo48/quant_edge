**原因诊断**：
- W9.3 的 "0 false alarm" 是 **window-level**（13 个 windows 的平均值），W9 + W11 这两个 weak windows 被 gate 识别
- 但实际 backtest 在 **date-level** 触发：495/2231 dates (22%) 命中 Rule B，集中在 2022-2025 (W6-W11 全期)
- 2022 年（W6, W7 末段）credit_spread 经常 < -1.5（信用利差紧），dispersion < 0.10，命中 gate
- 但 W7 (2022-09~2023-02) IC=+0.124 是 strong window，gate 切仓 50% 反而拖累
- **window-level 0-false-alarm ≠ date-level 0-false-alarm**

**结论**: Rule B as date-level overlay 是 **error of generalization**。W9.3 的 ex ante 信号在 aggregation level 上有效，但落到日频实施时频繁误触发。

## 重要发现

### 1. Score-weighted dominate equal-weight top-decile
EW top-decile 是 W7-W9 默认的 "academic" 组合方式，但在 production 维度下：
- EW 强制选 top 10%（50 stocks），无 buffer，每周大幅旋转
- 周换手率 24.1%, 年化 cost drag 6.9%
- gross excess +2.8% 全部被 cost 吃掉，net -4.1%

Score-weighted (top 20% + buffer):
- 用得分给权重，强信号给重仓
- buffer 大幅降低换手 → 2.65% 周换手
- net excess +7.3%，是 EW 的 3 倍以上

**经验**: 生产优化必须做，不能只用 academic IC 评估法。

### 2. Black-Litterman 在我们的设置下表现差
- selection_pct=0.20 + max_weight=0.10 → 集中度过高
- BL covariance lookback 60d → 噪声大
- 周换手 31%, cost drag 12.8% → 完全吞噬 alpha
- W22 优化版 BL 用了 sector/beta 约束，本 W10 简化版没启用

**W10 不推荐 BL**。如果要重新评估，需要 W11/W12 加上：
- Hierarchical risk parity 或 robust covariance
- Sector neutral / beta neutral 约束 (W22 cvxpy 框架)
- 更长 lookback (252d)

### 3. Rule B regime gate 不能直接用作 date-level overlay
- W9.3 给的 ex ante 信号有效（在 window 平均上）
- 但 date-level 实施会频繁误报（22% dates 命中）
- 切仓后牺牲 strong period 多于受益于 weak period
- 需要 W11 重新设计 regime detection（更精细 features 或 unsupervised approach）

### 4. 60D Ridge alone 是 production champion
- W7 推荐的 fusion / drop fundamental → W8 否决
- W8 的 LightGBM/XGB → 在干净数据上不如 Ridge
- W9 的 minute-proxy bonus → sampling artifact
- W10 的 BL/EW/regime gate → 都不如简单的 score-weighted Ridge
- **教训**：复杂方法在 PIT-fixed 数据上不一定胜出，简单 baseline 是真冠军
