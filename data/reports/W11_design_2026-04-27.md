## Codex 核心 finding

> **Democratic 4-way blend 在本数据集上大概率 destructive**（W7 教训）
>
> 唯一可行的设计是 **60D-dominant, bounded, asymmetric**：
> - 60D = 决定 admissible book
> - 20D = bounded rerank 在 60D envelope 内
> - 5D = sparse premium/penalty 仅触及极端
> - 1D = 调整今天 trade Δ，不动 score

**关键差别**：
- W7 失败 = symmetric score blend → 信号互相稀释
- W11 应该 = asymmetric control stack → 60D 主仓不动，其他三层只在 margin 调整

## 1. 推荐 fusion 公式

### 跨横截面 z-score normalization

每个 trade_date `t`，每 ticker `i`，每 horizon `h`:
```
z_h(t,i) = cross_sectional_zscore(p_h(t,i))   # p = Ridge 预测
```

### Layered overlay 公式

```
r60(t,i) = pct_rank(z60(t,i))
E_t = {i : r60(t,i) >= 0.65}                         # 60D envelope: top 35%

u(t,i) = z60(t,i)
       + 0.20 * clip(z20(t,i), -2, +2)               # 20D bounded adjust
       + 0.10 * sign(z5(t,i)) * 1{|z5(t,i)| >= 1.5}  # 5D sparse event
                * 1{r60(t,i) >= 0.50}                # 仅在 60D top half 触发

fused_score(t,i) = pct_rank(u(t,i) within E_t)       # 仅 envelope 内有效
                 = NaN otherwise

throttle_signal(t,i) = z1(t,i)                       # 1D 不动 score, 只调 Δw
```

### 1D throttle 实施（execution layer）

在 portfolio simulation 内：
```
buy with z1 < -1.0 (今天弱):  执行 50% intended Δw_increase
sell with z1 > +1.0 (今天强): 执行 50% intended Δw_decrease
其他: 全额执行
```

## 4. Pass Gate (Codex 修正版)

### 主条件 (1.0x cost, gate_off, vs W10 champion 7.34%/0.72/14.0%)

**Promote fusion if EITHER**:
- `net_ann_excess >= 7.84%` AND `IR >= 0.67`
- OR `net_ann_excess >= 7.09%` AND `IR >= 0.67` AND `max_dd <= 11.0%`

### Robustness check (1.25x cost)

- fused 仍 net_ann_excess > 5%
- AND fused 落后 60D <= 25 bps

### 不允许

- 不用 0.75x cost band 来 promote（cherry-picking optimistic cost）
- regime-gated fusion 救场不算 PASS（Rule B 已知 negative ROI）

## 5. Failure Mode + Fallback

| 场景 | 处置 |
|---|---|
| V1 (60D + 20D) 输给 V0 | 立即 STOP. 即 20D adjust 也 destructive. 无救. 关闭 W11 「no upgrade」 |
| V1 改善但 V2 / V3 恶化 | 至多 promote V1 (60D + 20D)，不上 V2/V3 |
| 所有 variants 输给 V0 | 关闭 W11，保 60D champion |
| V3 通过 strict gate 但 1.25x 不 robust | 不 promote，flag 为「optimistic-cost-dependent」 |

## 7. Codex Honest Assessment (概率)

| 概率 | 结果 |
|---|---|
| **10-20%** | symmetric blend 实施 → 大概率输 60D |
| **25-35%** | asymmetric design (above) → fusion >= 50bps 改善 |
| **35-45%** | fusion 持平 net 但 max_dd 改善 ≥ 3pp |
| **20-30%** | fusion 完全无 lift, V1 已 fail |

> **20D adjust 最有可能帮上忙**
> **5D 全 book rerank + 1D score blend 最可能 destructive**
> 所以正确的 W11 bet 不是「4 个弱信号一起打败 1 个强信号」
> 而是「1 个强信号在 1 个中信号 rerank + 2 个弱信号触及 edge case 后可能交易得稍好」
