## C2. Live IC Drift 阈值

**Backtest baseline**: mean_test_ic = **0.0699** (W9.1 13-window OOS)

| 状态 | 4-week rolling live IC | 操作 |
|---|---|---|
| 🟢 Green | ≥ 0.04 | 正常运行, 不动 sizing |
| 🟡 Yellow | 0.02 - 0.04 | Alert + 观察 (允许 2 周) |
| 🔴 Red | < 0.02 持续 2 周 | **降仓 50%**, 启动 root cause review |
| ⚫ Black | < 0 持续 4 周 | **完全平仓**, 停 greyscale, 重做研究 |

**Drift Decay Ratio** = live_ic / backtest_ic (0.0699):
- ≥ 0.50 (= 0.035): green
- 0.25-0.50 (0.017-0.035): yellow alert
- < 0.25 (0.017): red

**Implementation**: 由 `scripts/run_greyscale_monitor.py` 每 4 周计算, output 到 `data/reports/greyscale/monitor_*.json`.

**Note**: 前 4 周 live IC 不成熟 (label horizon 60D，至少要 8-12 周 calendar 才能算"forward 60D 实际收益"). 用 5D / 20D 短期指标作 leading indicator.

## C5. Layer 3 Red Alert 处置

**Backtest baseline**: 周换手 2.65%, max single weight 5%, holdings 20+, sector deviation < 0.10

| Layer 3 Red 触发条件 | 处置 |
|---|---|
| holdings count < 15 | 🔴 **不下新单 + 持仓**, 人工 review (可能信号 collapse) |
| 单 ticker weight > 0.05 (max_weight 突破) | 🔴 数据/约束 bug, 立即停 + 修 |
| 周换手 > 0.20 (8x backtest 期望) | 🔴 alert + 持仓 (不强制平仓), 验证 turnover_buffer 配置 |
| Cash weight > 0.25 | 🔴 alert + 持仓 (信号源 dispersion 过低 / 信号不可信) |
| Sector deviation > 0.15 | 🔴 alert + 持仓, 验证 sector map 是否 stale |
| CVaR_99 < -0.05 | 🔴 风险事件, alert + 持仓, oncall 决定下一步 |

**统一原则**:
- W12 prep + 第一个 4 周 greyscale 期间, **所有 red alert = hold_positions_and_alert** (不自动平仓不切 SPY)
- **manual_clear_required: true** (alert 不自动 clear, 必须人工标 fixed)
- 若 4 周内连续 ≥ 2 次 red alert, **自动停 greyscale**, 重启需重新 W12 prep

## 启动 Calendar Greyscale 命令

确认所有硬条件后, 启动每周 Friday paper trading:

```bash
source .venv/bin/activate && \
ulimit -v 14000000 && \
taskset -c 0-3 nice -n 19 \
python scripts/run_greyscale_live.py \
  --bundle-path data/models/bundles/w12_60d_ridge_swbuf_v1/bundle.json \
  --report-dir data/reports/greyscale \
  --reference-feature-matrix-path data/features/walkforward_v9full9y_fm_60d.parquet
```

**注意**:
- `--report-dir` 是 `data/reports/greyscale/` (生产目录，不是 dryrun 隔离目录)
- 不带 `--dry-run` = 真实保存 weekly report (但无真钱交易)
- 不带 `--limit-tickers` = full universe
- 不带 `--as-of` = 自动取当前 PIT-visible 最新 trade_date

可由 cron / Airflow DAG 每周 Friday 18:00 ET 触发.
